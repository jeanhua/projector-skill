## Usage Guide

### Starting a New Conversation

1. Read this README for project overview
2. Check `architecture/overview.md` for system architecture
3. Review relevant module documentation based on your task

### Before Modifying Code

1. Consult the corresponding module documentation in `modules/`
2. Check `patterns/reusable.md` for reusable code
3. Review `patterns/conventions.md` for coding conventions
4. Check `issues/known-issues.md` for related known issues

### When Encountering Issues

1. Check `issues/resolved.md` for known solutions
2. Review `issues/workarounds.md` for temporary solutions
3. Check `context/decisions.md` for related technical decisions

### When Writing New Code

1. Follow coding conventions in `patterns/conventions.md`
2. Prioritize reusing existing code from `patterns/reusable.md`
3. Avoid anti-patterns listed in `patterns/anti-patterns.md`
4. Add newly discovered reusable code to `patterns/reusable.md`

### After Solving Issues

1. Add the issue and solution to `issues/resolved.md`
2. If it's a general solution, consider adding to `patterns/reusable.md`
3. Update relevant module documentation

## Updating the Knowledge Base

### Automatic Updates

The project is configured with Git hooks that automatically detect changes and prompt for updates after each commit.

### Manual Updates

Run `/projector update` to manually update the knowledge base.

### What to Update

- Module documentation: When module code changes
- Reusable code: When new reusable code is discovered
- Known issues: When new issues are found or resolved
- Architecture: When significant architectural changes occur
